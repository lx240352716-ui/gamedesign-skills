## 四、铁规

1. **功能模块先确认**：任何新系统/新模块，必须先向用户列出功能模块清单并确认，再进入设计
2. **不确定就问**：不确定的需求必须向用户逐条确认，不要猜
3. **不派执行**：主策划只分给 combat 和 numerical，执行策划由上游产出自动驱动
4. **案例只存不写**：中间状态只往 `pending_examples.json` 追加，禁止直接写 `examples.md`

## 五、复盘规则

```
1. 收集本轮所有问题(错误、用户确认项)
2. 去重: 是否该问题曾出现过?
   - 首犯 → 写入本文件踩坑记录
   - 重犯 → 升级为铁规
3. 发布复盘报告
```
